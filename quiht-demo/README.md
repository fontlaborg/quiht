# quiht-demo

Source for the static demo at **https://fontlab.org/quiht/**. It lets anyone
drag-and-drop a Qt `.ui`, `.quiht.json`, or `.quiht.zip` and see it rendered via
[`quiht-core`](https://www.npmjs.com/package/quiht-core) — entirely client-side,
no server.

The build outputs into the repository's top-level `../docs/` directory, which is
served by GitHub Pages. `base: "./"` keeps every asset path relative so the page
works under any sub-path.

```bash
npm install
npm run dev      # local dev server
npm run build    # type-check + build into ../docs
npm run preview  # preview the built output
```

`public/sample.quiht.zip` (the "Try a sample bundle" dataset) is generated by
`build.sh` from `../example/` using `quiht-tools`; it is not committed here.
